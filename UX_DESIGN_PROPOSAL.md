### 1. **Quick-Scan Capability**
Practitioners need to glance at instructions during live sessions. Current dense paragraphs make this difficult.

### 2. **Visual Breathing Cues**
Inspired by Headspace/Calm: Subtle visual indicators help practitioners pace their delivery.

### 3. **Professional Typography Hierarchy**
Clear distinction between:
- **Script phrases** (what to say verbatim)
- **Timing cues** (when to pause)

- **Step-by-step instructions**

### 4. **Ample White Space**
Mindfulness.com style: Generous spacing reduces cognitive load and creates calm.

### **A. Instruction Formatting Enhancement**

**Current Issue:** Instructions are dense paragraphs, hard to scan during sessions.

**Proposed Solution:**
1. **Parse instructions into structured blocks:**
   - **Script phrases** (highlighted, larger font) - what to say
   - **Timing markers** (subtle, italic) - pause indicators
   - **Step numbers** (bold, colored) - clear progression
   - **Body cues** (smaller, secondary color) - practitioner notes

3. **Typography hierarchy:**
   ```
   SCRIPT PHRASE (24px, bold, navy) - What to say
   Step number (18px, purple accent) - Navigation
   Timing cue (16px, italic, gray) - Pacing
   Practitioner note (14px, secondary) - Internal guidance
   ```

### **B. Enhanced Exercise Detail View**

**Improvements:**
1. **Sticky header** with exercise name and duration (always visible)
2. **Quick reference bar** at top: Duration | Competency | Validation level
3. **Collapsible sections** (all start collapsed except Instructions)
4. **"Quick Start" mode** - shows only script phrases
5. **Visual progress** through steps (subtle progress indicator)

### **C. Visual Design Refinements**

1. **Spacing:**
   - Increase line-height to 1.8-2.0 for instructions
   - Add 24px spacing between major sections
   - 16px spacing between steps

2. **Color Psychology:**
   - Keep current palette (calming, professional)
   - Add subtle accent for script phrases (slightly warmer tone)
   - Use muted gray for timing cues (non-intrusive)

3. **Typography:**
   - Increase instruction font size to 20px (mobile) / 22px (desktop)
   - Use system font stack for better readability
   - Add letter-spacing: 0.3px for better legibility

### **D. Practitioner Confidence Features**

1. **"Script Mode" toggle:**
   - Shows only what to say (hides internal notes)
   - Larger font, centered text
   - Minimal distractions

2. **Timing helper:**
   - Optional timer for exercises
   - Visual countdown for pauses
   - Subtle vibration cues (if supported)

3. **Quick navigation:**
   - Swipe between exercises
   - Jump to specific steps
   - Bookmark favorite phrases

## üîç **QUESTIONS FOR CLARIFICATION**

Before implementing, I need to understand:

1. **Use Case Priority:**
   - Do practitioners read instructions WHILE leading (quick glances)?
   - Or do they prepare BEFORE sessions (full read-through)?
   - Or both?

2. **Script vs. Guidance:**
   - Should instructions be word-for-word scripts to read?
   - Or guidance on what to convey (practitioner adapts language)?
   - Mix of both?

3. **Timing Precision:**
   - Do you need exact timing cues ("pause 30 seconds")?
   - Or general pacing ("take your time here")?
   - Should there be a built-in timer?

4. **Visual Preferences:**
   - Subtle breathing animations (like Headspace)?
   - Or keep it minimal/text-only?
   - Any specific visual metaphors you want?

5. **Content Priority:**
   - What's most important during a live session?
     - Script phrases?
     - Timing?
     - Participant guidance?
     - Troubleshooting tips?

6. **Device Context:**
   - Phone on table/stand during session?
   - Phone in hand (quick glances)?
   - Tablet for larger display?
